##=================================================
#PROPOSITO
Este trabajo esta orientado a simular el proceso 
creativo de los humanos a través de herramientas 
de I.A. (ChatGPT y Mermaid AI), tratando de 
encontrar soluciones novedosas a un desafío 
específico de JavaScript. En este caso, funciones 
flecha y refactorización a través de ellas 
(programación modular).
##=================================================
##FASE 1 - Identificación y pensamiento divergente
##=================================================
Se le pidió a la I.A. asociar las funciones 
flecha, programación modular y refactorización 
con conceptos de motocicletas (deportivas y 
urbanas). Aplicando el protocolo TRACI en el 
primer prompt, y posteriormente el protocolo 
ToT para filtrar los resultados. Seleccioné la 
opción 3 ya que la I.A. lo consideraba la menos 
optima, y dado que había preseleccionado la opción 
2, este resultado podría tener sesgo de elección 
implicito.
##=================================================
##FASE 2 - Desarrollo y desafio del sesgo
##=================================================
Se le pidió a la I.A. usar una estrategía 
contraintuitiva y sugerió utilizar la 
desmodulación como método de aprendizaje, ya que 
al ir desarmando la moto, vas aprendiendo que 
herramientas (funciones y atributos) desamblan 
los componentes (clases, metodos y objetos), y 
de esta manera puedes volver a construir 
nuevamente tu moto (modulación). La I.A. lo 
nombró "top-down coding".
##=================================================
##FASE 3 - Revisión y refinamiento/evaluación
##=================================================
Se le solicitó a la I.A. refinar su propuesta y 
mencionó que bajo condiciones simples, todo se 
ejecutaría de manera adecuada. Aborda temas 
como la necesidad de parametrización (funciones 
como atributos de objetos para prescindir del 
"hard code"), manejo de excepciones y el uso 
de convenciones como el camelCase.

Sugiere utilizar /** */ para documentar funciones 
criticas y hacer entendible el codigo. 

Dentro de sus optimizaciones, la I.A. utiliza 
la metodología SOLID para separar las funciones, 
refactorizar y facilitar los testeos.
